HTML (para o eu do futuro)

Mano, HTML é basicamente o esqueleto do site. É ele que define o que tem na página: título, texto, imagem, botão, tudo. Não tem lógica nem estilo, é só estrutura mesmo. Pensa como se fosse montar um corpo sem pele nem roupa, só os ossos organizando tudo.

CSS (para o eu do futuro) 

Aqui é onde o site deixa de ser feio. CSS serve pra estilizar tudo: cor, tamanho, posição, animação, essas coisas. Se o HTML é o esqueleto, o CSS é a aparência, tipo roupa e estética. Sem isso, o site funciona, mas fica horroroso.

JS (para o eu do futuro) 

JavaScript é o que faz o site ter vida. Clique, botão funcionando, validação de formulário, animação mais complexa, tudo vem daqui. É a lógica do negócio. Se HTML é estrutura e CSS é aparência, o JS é o cérebro que faz tudo acontecer.

Diferenças entre Java, JS (JavaScript) e TS (TypeScript)

Java e JavaScript não são a mesma coisa, apesar do nome parecido, são linguagens completamente diferentes. Java é mais usado pra sistemas grandes, backend e apps mais robustos. JavaScript roda principalmente no navegador e é focado em web. TypeScript é tipo um “JavaScript melhorado”, com tipagem, que ajuda a evitar erro e organizar melhor o código.

-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

POO (Programação Orientada a Objetos)

POO é um jeito de programar organizando tudo como se fossem “objetos” da vida real. A ideia é deixar o código mais organizado e reutilizável. Em vez de sair fazendo tudo solto, você cria estruturas com responsabilidades. Isso ajuda muito quando o projeto cresce.

-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

Classe

Classe é tipo um molde. Ela define como um objeto vai ser, quais atributos e métodos ele tem. Não é o objeto em si, é só a base. É como uma planta de uma casa antes de construir.

-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

Objeto

Objeto é a instância da classe, ou seja, é quando você cria algo real baseado no molde. Ele já tem valores e pode executar ações. É a casa pronta baseada na planta. No código, é o que você realmente usa.

-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

Atributo

Atributo é uma característica do objeto. Tipo nome, idade, cor, etc. Ele guarda informação. É basicamente uma variável dentro da classe.

-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

Método

Método é uma ação que o objeto pode fazer. Tipo andar, atacar, calcular, mostrar algo. É uma função dentro da classe. Serve pra dar comportamento ao objeto.

-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

Getters e Setters

Getters e setters são formas controladas de acessar e alterar os dados. Getter pega o valor, setter muda o valor. Isso ajuda a proteger o código e evitar bagunça. Não é obrigatório, mas é uma boa prática.

-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

Construtor

Construtor é um método que roda automaticamente quando você cria um objeto. Ele serve pra já definir valores inicais. É tipo configurar tudo na hora que nasce. Facilita muito pra não esquecer nada.

-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

Herança

Herança é quando uma classe herda coisas de outra. Você reaproveita código sem precisar copiar. A classe “filha” ganha atributos e métodos da “mãe”. Ajuda a manter tudo mais organizado.

-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

Encapsulamento

Encapsulamento é esconder os detalhes internos e controlar o acesso. Você não deixa qualquer parte do código mexer em tudo. Isso evita erro e bagunça. É tipo proteger o que é importante.

-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

Polimorfismo

Polimorfismo é quando algo pode ter várias formas. No código, é quando métodos com o mesmo nome funcionam de maneiras diferentes. Isso deixa o código mais flexível. Você usa a mesma ideia, mas com comportamentos diferentes dependendo do contexto.
